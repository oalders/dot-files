# `bin/merge-pr` Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `bin/merge-pr` — a single-command "merge this PR and tear down the worktree, branch, and tmux session" tool, run from inside the PR's worktree.

**Architecture:** A self-contained bash script in `bin/` (already on `$PATH` via `bashrc`). Wraps `gh pr merge`, then performs cleanup in an order that survives killing the tmux session it's running inside. Argument-parses our own `-f`/`--force`, refuses `--auto`/`--auto-merge`, passes everything else through to `gh pr merge`.

**Tech Stack:** bash 4+, `gh` CLI, `git` (incl. worktree commands), `tmux`. Bats (already installed at `node_modules/.bin/bats`, v1.13.0) for tests. No new dependencies.

**Reference spec:** `docs/superpowers/specs/2026-05-04-merge-pr-design.md`.

**Testing approach:** Bats wires up here for the first time in this repo. Tests live in `test/`. Bats covers high-leverage areas: arg parsing/refusal (pure logic) and pre-check refusal messages (testable against a git tempdir with a stubbed `gh`). The genuinely-stateful end-to-end scenarios (live `gh pr merge`, tmux session kill) stay in the manual test matrix at Task 6.

- [ ] **Step 1: Create the script with skeleton + arg parsing**

Create `bin/merge-pr` with the following content:

```bash
#!/usr/bin/env bash

set -eu -o pipefail

usage() {
    cat <<'EOF'
Usage: merge-pr [-f|--force] [gh pr merge args...]

Merges the current branch's PR via `gh pr merge`, then removes the
worktree, deletes the local branch, and kills the tmux session for the
worktree. Run from inside the worktree's tmux session.

Options:
  -f, --force   Pass --force to `git worktree remove` (discards
                uncommitted changes in the worktree).
                May appear anywhere in the argument list.

Refuses:
  --auto, --auto-merge   gh pr merge would exit 0 while the PR stays
                         OPEN, and our cleanup would tear down the
                         worktree before the merge actually happens.
EOF
}

# Parse our own --force/-f from anywhere in the argument list.
# Refuse --auto / --auto-merge.
force=""
gh_args=()
for arg in "$@"; do
    case "$arg" in
        -f | --force)
            force="--force"
            ;;
        --auto | --auto-merge)
            echo "merge-pr: refusing $arg — cleanup would run before the merge completes" >&2
            exit 2
            ;;
        -h | --help)
            usage
            exit 0
            ;;
        *)
            gh_args+=("$arg")
            ;;
    esac
done
```

- [ ] **Step 1: Add the pre-flight check block to the script**

Append to `bin/merge-pr` (after the arg-parsing `for` loop):

```bash
# Pre-flight: must be inside a git work tree.
if ! git rev-parse --is-inside-work-tree >/dev/null 2>&1; then
    echo "merge-pr: not inside a git work tree" >&2
    exit 1
fi

branch=$(git rev-parse --abbrev-ref HEAD)

# Pre-flight: branch must have an upstream.
if ! git rev-parse --abbrev-ref --symbolic-full-name '@{u}' >/dev/null 2>&1; then
    echo "merge-pr: branch '$branch' has no upstream — push first" >&2
    exit 1
fi

# Pre-flight: no unpushed commits.
unpushed=$(git rev-list --count '@{u}..HEAD')
if [[ "$unpushed" -ne 0 ]]; then
    echo "merge-pr: branch '$branch' has $unpushed unpushed commit(s) — push first" >&2
    exit 1
fi
```

(The "branch is not the base branch" check is intentionally NOT here — it requires the PR's `baseRefName`, which we fetch in Task 4 along with the PR state, to avoid a redundant `gh` round-trip.)

```bash
# Set up a fresh git repo with an initial commit in BATS_TEST_TMPDIR/repo
# and cd into it. After this, HEAD is on a branch with one commit and
# no remote configured.
setup_git_repo() {
    REPO_DIR="$BATS_TEST_TMPDIR/repo"
    mkdir -p "$REPO_DIR"
    cd "$REPO_DIR"
    git init -q -b main
    git config user.email "test@example.com"
    git config user.name "Test"
    echo "x" >file
    git add file
    git -c commit.gpgsign=false commit -q -m "init"
}

# Set up a "bare upstream" simulating a remote, and configure tracking.
# After this, the current branch is pushed to "origin" and tracking is set.
setup_upstream() {
    UPSTREAM_DIR="$BATS_TEST_TMPDIR/upstream.git"
    git init --bare -q "$UPSTREAM_DIR"
    git remote add origin "$UPSTREAM_DIR"
    git push -q -u origin HEAD
}
```

- [ ] **Step 1: Append state capture and PR-state dispatch to the script**

Append to `bin/merge-pr`:

```bash
# State capture (before any mutation).
worktree_path=$(git rev-parse --show-toplevel)
git_common_dir=$(git rev-parse --git-common-dir)
# In a linked worktree, --git-common-dir returns the main repo's .git;
# its parent is the main repo working directory.
main_repo=$(cd "$git_common_dir" && cd .. && pwd)

# PR lookup: state + baseRefName in a single call.
if ! pr_info=$(gh pr view --json state,baseRefName -q '[.state, .baseRefName] | @tsv' 2>/dev/null); then
    echo "merge-pr: no PR found for branch '$branch'" >&2
    exit 1
fi
pr_state=$(echo "$pr_info" | cut -f1)
pr_base=$(echo "$pr_info" | cut -f2)

# Pre-flight (deferred from Task 3): current branch must not be the PR's base.
if [[ "$branch" == "$pr_base" ]]; then
    echo "merge-pr: refusing to merge from base branch '$pr_base'" >&2
    exit 1
fi

# Dispatch on PR state.
case "$pr_state" in
    OPEN)
        gh pr merge "${gh_args[@]}"
        ;;
    MERGED)
        echo "merge-pr: PR already MERGED — proceeding to cleanup"
        ;;
    *)
        echo "merge-pr: refusing to act on PR in state '$pr_state'" >&2
        exit 1
        ;;
esac
```

- [ ] **Step 1: Add session resolution**

Edit `bin/merge-pr`. Locate the line:

```bash
# Dispatch on PR state.
```

Insert this block immediately before it:

```bash
# Tmux session resolution. Done before mutation so a later cd doesn't
# affect the lookup. Empty string means "no session to kill".
session=""
if [[ -n "${TMUX:-}" ]]; then
    session=$(tmux display-message -p '#S')
elif command -v tmux >/dev/null 2>&1 && tmux list-panes -a >/dev/null 2>&1; then
    # Fallback: find a pane whose start-path matches the worktree.
    # We use pane_start_path (not session_path or pane_current_path)
    # because it reflects where tmux launched the pane and doesn't
    # drift if the user cd's.
    session=$(
        tmux list-panes -a -F '#{session_name}	#{pane_start_path}' |
            awk -F '\t' -v wp="$worktree_path" '$2 == wp {print $1; exit}'
    )
fi
```

### Task 6: Cleanup sequence

**Files:**
- Modify: `bin/merge-pr` (append after the state-dispatch `case` block)

- [ ] **Step 1: Append the cleanup sequence**

Append to `bin/merge-pr`:

```bash
# Cleanup. Order matters:
#   1. cd out of the worktree before removing it.
#   2. Remove the worktree.
#   3. Delete the local branch (forced because squash/rebase merges
#      aren't recognized as merged by `git branch -d`).
#   4. Kill the tmux session LAST — when invoked from inside that
#      session, this terminates the script. Nothing may follow.
cd "$main_repo"

# shellcheck disable=SC2086
# $force is intentionally unquoted: empty string must expand to nothing,
# matching the pattern used in bin/add-worktree.
if ! git worktree remove $force "$worktree_path"; then
    echo "" >&2
    echo "merge-pr: worktree has uncommitted changes." >&2
    echo "          Re-run with: merge-pr --force" >&2
    exit 1
fi

git branch -D "$branch"

if [[ -n "$session" ]]; then
    tmux kill-session -t "$session"
fi
```
