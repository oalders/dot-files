# Dev Container Rework: Token Auth, Minimal Mounts, Firewall

## Problem

The current `bin/dev` setup mounts several host files for Claude authentication
(`~/.claude.json`, `~/.claude/.credentials.json`, `~/.claude/settings.json`) and
uses a host directory (`~/.claude-containers/`) for session persistence. This
creates coupling between host and guest, and the host-managed session directories
are prone to accidental deletion.

## Goals

1. Partition the container from the host filesystem as much as possible
2. Use token-based auth (`claude setup-token`) instead of mounted credential files
3. Use Docker named volumes for session persistence (host can't interfere)
4. Apply network firewall restrictions when running Claude
5. Keep the image self-contained ā€” rebuild nightly for updated plugins/settings

## Approach

Rework the existing `bin/dev` script in-place. Same command interface, new
internals for auth, mounts, and firewall.

## Authentication

Replace mounted credential files with a `CLAUDE_TOKEN` environment variable.

- Resolution order: `CLAUDE_TOKEN` env var, then `~/.claude-token` file
- Passed to Docker as `--env CLAUDE_TOKEN=...`
- Error with clear message if neither source provides a token
- **Removed mounts:** `~/.claude.json`, `~/.claude/.credentials.json`

The token is generated by `claude setup-token` ā€” a long-lived OAuth token (valid
~1 year) tied to the user's Pro/Max subscription.

**Note:** The exact env var Claude expects inside the container needs to be
verified. `claude setup-token` documentation should specify whether it's
`CLAUDE_TOKEN`, `CLAUDE_CODE_OAUTH_TOKEN`, or another name.

## Mounts

Three mounts total, down from six-seven:

| Mount | Source | Target | Purpose |
|-------|--------|--------|---------|
| Project dir | `$(pwd)` | `/workspace` | Project files |
| Git common dir | Absolute host path | Same absolute path | Worktree support |
| Session volume | Named volume `dev-<repo>-<branch>-sessions` | `/home/agent/.claude/projects` | Session persistence + history |

### Removed mounts

- `~/.claude.json` ā€” replaced by env var
- `~/.claude/.credentials.json` ā€” replaced by env var
- `~/.claude/settings.json` ā€” baked into image
- `~/.claude-containers/<name>/` ā€” replaced by named volume
- `~/.claude-containers/<name>/history.jsonl` ā€” lives inside named volume

### Git worktree support

Unchanged from current implementation. When the working directory is a git
worktree, the main `.git` directory is detected via `git rev-parse
--git-common-dir` and mounted at its absolute host path so git references
resolve correctly inside the container.

## Session persistence

Docker named volumes replace the host directory approach.

- Volume naming: `dev-<repo>-<branch>-sessions`
- Created automatically on first `dev start`
- Persists across container stop/start/rebuild
- `dev stop` removes the container but keeps the volume
- Docker manages storage ā€” host processes cannot accidentally delete sessions

History (`history.jsonl`) lives inside the named volume alongside session data.

## Firewall

Reuse the existing `claude-firewall.sh` (iptables whitelist) with these changes:

- Container starts with `--cap-add=NET_ADMIN` to allow iptables modification
- Firewall runs inline before Claude in `dev claude` command only
- `dev shell` does NOT run the firewall ā€” unrestricted access for debugging

Implementation in `dev claude`:

```bash
docker exec <container> bash -c '/usr/local/bin/claude-firewall.sh && exec claude "$@"' _ "$@"
```

### Whitelisted traffic (from existing claude-firewall.sh)

- DNS (UDP 53)
- SSH (TCP 22)
- Localhost
- Anthropic APIs (api, auth, console, statsig.anthropic.com)
- Sentry, Statsig
- npm registry
- GitHub (github.com, api.github.com, dynamic IP ranges)

## Image changes (Dockerfile.dev)

- Bake `settings.json` into the image at `/home/agent/.claude/settings.json`
- No entrypoint changes
- Nightly rebuilds keep plugins, tools, and settings current
- Everything else in Dockerfile.dev remains unchanged

## Command interface

Existing commands unchanged in interface. Two new commands added:

| Command | Behavior |
|---------|----------|
| `dev build` | Build image (unchanged) |
| `dev start` | Start container with named volume, `NET_ADMIN` cap |
| `dev shell` | Interactive shell, no firewall |
| `dev claude [args]` | Run firewall then Claude |
| `dev reclaude [args]` | Stop and restart with Claude |
| `dev stop` | Stop/remove container, keep volume |
| `dev status` | Show container status (unchanged) |
| `dev clean` | **New.** Remove session volume for current repo+branch |
| `dev clean --all` | **New.** Remove all `dev-*-sessions` volumes |

## Volume lifecycle

```
dev start    ā†’ container created, volume created (if not exists)
dev stop     ā†’ container removed, volume kept
dev clean    ā†’ volume removed for current project
dev clean --all ā†’ all dev-*-sessions volumes removed
```

## Migration

- The `~/.claude-containers/` directory becomes unused after this change
- Users should generate a token via `claude setup-token` and save to
  `~/.claude-token` or set `CLAUDE_TOKEN` env var
- No backward compatibility shims ā€” clean break
